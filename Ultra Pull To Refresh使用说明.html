Ultra Pull To Refresh 下拉刷新 上拉加载

Ultra Pull To Refresh 下载地址： https://github.com/liaohuqiu/android-Ultra-Pull-To-Refresh

一 功能介绍

android-Ultra-Pull-To-Refresh提供了一个下拉刷新、上拉加载的技术框架。继承于ViewGroup可以包含任何View。功能比SwipeRefreshLayout强大。支持所有的View: ListView, GridView, RecycleView，ScrollView, FrameLayout, 甚至 TextView.

二 使用方法

1、添加依赖：

 compile 'in.srain.cube:ultra-ptr:1.0.11'   //只有下拉刷新
 compile 'in.srain.cube:ptr-load-more:1.0.2'  //下拉上拉均有

2、配置xml的布局文件：

有6个参数可配置

阻尼系数：默认: 1.7f，越大，感觉下拉时越吃力。

触发刷新时移动的位置比例：默认，1.2f，移动达到头部高度1.2倍时可触发刷新操作。

回弹延时：默认 200ms，回弹到刷新高度所用时间

头部回弹时间：默认1000ms

刷新是保持头部：默认值 true.

下拉刷新 / 释放刷新：默认为释放刷新

<in.srain.cube.views.ptr.PtrClassicFrameLayout
    android:id="@+id/pullToRefresh_layout"
    xmlns:cube_ptr="http://schemas.android.com/apk/res-auto"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    cube_ptr:ptr_duration_to_close="200"
    cube_ptr:ptr_duration_to_close_header="1000"
    cube_ptr:ptr_keep_header_when_refresh="true"
    cube_ptr:ptr_pull_to_fresh="false"
    cube_ptr:ptr_ratio_of_header_height_to_refresh="1.2"
    cube_ptr:ptr_resistance="1.7"> //阻尼系数

也可以在代码中定义

mPtrFrame.setResistance(1.7f);
mPtrFrame.setRatioOfHeaderHeightToRefresh(1.2f);
mPtrFrame.setDurationToClose(200);
mPtrFrame.setDurationToCloseHeader(1000);    
mPtrFrame.setPullToRefresh(false); //默认是false     
mPtrFrame.setKeepHeaderWhenRefresh(true); //默认是true

3、设置下拉刷新上拉加载组件的事件监听：

调用mPtrFrame.setPtrHandler()方法来实现我们需要的模式。

上拉下拉均有的模式： 
mPtrFrame.setPtrHandler(new PtrDefaultHandler2() {
//重写方法忽略
}

只有下拉的模式：
mPtrFrame.setPtrHandler(new PtrDefaultHandler() {
//重写方法忽略
}

例如：

下拉刷新、上拉加载均有的模式

mPtrFrame.setPtrHandler(new PtrDefaultHandler2() {
    @Override
    public void onLoadMoreBegin(final PtrFrameLayout frame) {
        mPtrFrame.postDelayed(new Runnable() {
            @Override
            public void run() {

                frame.refreshComplete();
            }
        }, 1000);
    }

    @Override
    public void onRefreshBegin(final PtrFrameLayout frame) {
        mPtrFrame.postDelayed(new Runnable() {
            @Override
            public void run() {

                frame.refreshComplete();
            }
        }, 1000);
    }
});

只有下拉刷新的模式

    mPtrFrame.setPtrHandler(new PtrDefaultHandler() {
        @Override
        public void onRefreshBegin(PtrFrameLayout frame) {
            mPtrFrame.postDelayed(new Runnable() {
                @Override
                public void run() {
                    mPtrFrame.refreshComplete();
                }
            },1000);
        }
    });

4、下拉时头部的实现风格：

1、PtrClassicDefaultHeader 风格（经典）

PtrClassicDefaultHeader 实现了 PtrUIHandler 接口。

如图：

配置信息如下：

  //默认经典头布局
PtrClassicDefaultHeader header =new PtrClassicDefaultHeader(this);

 //默认经典底布局
PtrClassicDefaultFooter footer=new PtrClassicDefaultFooter(this);

 mPtrFrame.setHeaderView(header);
 mPtrFrame.addPtrUIHandler(header);
2、StoreHouse 风格

使用字符串, 支持A-Z, 0-7 以及 - .等

如图：

配置信息如下：

       //闪动文字的头布局
    StoreHouseHeader header = new StoreHouseHeader(this);
    header.setPadding(0, PtrLocalDisplay.dp2px(20), 0, 0);

    //使用资源文件中的路径信息
   // header.initWithStringArray(R.array.refresh_color);

   // 设置加载数据时显示的字符串，文本只支持0-9，英文字符
    header.initWithString("xiala",20);
    header.setDropHeight(300);
    header.setBackgroundColor(Color.BLACK);
    header.setTextColor(android.R.color.holo_blue_dark);

    mPtrFrame.setHeaderView(header);
    mPtrFrame.addPtrUIHandler(header);
